# Linux 开机启动部署指南

本文档说明如何在 Linux 系统上使用 systemctl 或 nohup 配置服务器端和客户端开机自启动。

## 一、创建工作目录

```bash
# 创建应用目录
sudo mkdir -p /opt/monitor
sudo mkdir -p /var/log/monitor

# 将编译好的程序复制到目录
sudo cp server-linux-amd64 /opt/monitor/server
sudo cp client-linux-amd64 /opt/monitor/client

# 设置执行权限
sudo chmod +x /opt/monitor/server
sudo chmod +x /opt/monitor/client

# 创建数据目录
sudo mkdir -p /opt/monitor/data
```

## 二、服务器端服务配置

### 2.1 使用 nohup 启动（推荐用于简单部署）

```bash
# 创建日志目录
mkdir -p /var/log/monitor

# 使用 nohup 启动服务
nohup /opt/monitor/server --port 8080 --url systemmonitor/v1 -quiet > /var/log/monitor/server.log 2>&1 &
# 可以重定向到/dev/null 表示不需要日志

# 查看进程
ps aux | grep server

# 查看日志
tail -f /var/log/monitor/server.log
```

#### nohup 启动参数说明

| 参数 | 说明 |
|-----|------|
| --port | 服务器监听端口 |
| --url | URL 前缀路径 |
| --quiet | 静默模式，不输出日志 |

#### 停止 nohup 进程

```bash
# 查找进程 PID
ps aux | grep "server --port"

# 杀掉进程
kill <PID>
```

#### 添加到开机启动

在 `/etc/rc.local` 中添加（确保文件有执行权限）：

```bash
#!/bin/bash
# 在 exit 0 之前添加
nohup /opt/monitor/server --port 8080 --url systemmonitor/v1 -quiet > /var/log/monitor/server.log 2>&1 &
```

或者使用 crontab：

```bash
# 编辑 crontab
crontab -e

# 添加以下行（系统启动后 1 分钟执行）
@reboot sleep 60 && nohup /opt/monitor/server --port 8080 --url systemmonitor/v1 -quiet >> /var/log/monitor/server.log 2>&1 &
```

### 2.2 使用 systemctl（推荐用于生产环境）

#### 创建服务文件

```bash
sudo vim /etc/systemd/system/monitor-server.service
```

添加以下内容：

```ini
[Unit]
Description=System Monitor Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/monitor
ExecStart=/opt/monitor/server --port 8080 --url systemmonitor/v1 -quiet
Restart=always
RestartSec=10

# 日志配置
StandardOutput=append:/var/log/monitor/server.log
StandardError=append:/var/log/monitor/server.log

[Install]
WantedBy=multi-user.target
```

#### 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl start monitor-server
sudo systemctl status monitor-server
sudo systemctl enable monitor-server
```

## 三、客户端服务配置

### 3.1 使用 nohup 启动

```bash
# 创建日志目录
mkdir -p /var/log/monitor

# 启动客户端（单个服务器）
nohup /opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put > /var/log/monitor/client.log 2>&1 &

# 启动客户端（多个服务器）
# 向多个服务器发送相同的数据
nohup /opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put http://192.168.1.101:8080/systemmonitor/v1/put > /var/log/monitor/client.log 2>&1 &

# 参数说明：
# --id: 客户端识别码，可自定义
# --interval: 发送间隔（秒），默认60秒
# URL: 服务器地址（支持多个，用空格分隔）
```

#### 停止 nohup 进程

```bash
# 查找进程 PID
ps aux | grep "client --id"

# 杀掉进程
kill <PID>
```

#### 添加到开机启动

使用 crontab：

```bash
crontab -e

# 添加以下行（单个服务器）
@reboot sleep 30 && nohup /opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put >> /var/log/monitor/client.log 2>&1 &

# 添加以下行（多个服务器）
@reboot sleep 30 && nohup /opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put http://192.168.1.101:8080/systemmonitor/v1/put >> /var/log/monitor/client.log 2>&1 &
```

### 3.2 使用 systemctl

#### 创建服务文件

```bash
sudo vim /etc/systemd/system/monitor-client.service
```

添加以下内容：

```ini
[Unit]
Description=System Monitor Client
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/monitor
# 单个服务器
ExecStart=/opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put

# 多个服务器
ExecStart=/opt/monitor/client --id my-server-01 --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put http://192.168.1.101:8080/systemmonitor/v1/put
Restart=always
RestartSec=10

StandardOutput=append:/var/log/monitor/client.log
StandardError=append:/var/log/monitor/client.log

[Install]
WantedBy=multi-user.target
```

#### 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl start monitor-client
sudo systemctl status monitor-client
sudo systemctl enable monitor-client
```

### 3.3 启动多个客户端

如果需要在一台机器上运行多个客户端实例（向同一服务器发送数据）：

```bash
# 多个客户端使用不同的 --id
nohup /opt/monitor/client --id node1 --interval 30 http://192.168.1.100:8080/systemmonitor/v1/put > /var/log/monitor/client-node1.log 2>&1 &
nohup /opt/monitor/client --id node2 --interval 30 http://192.168.1.100:8080/systemmonitor/v1/put > /var/log/monitor/client-node2.log 2>&1 &

# 或者一个客户端向多个服务器发送数据
nohup /opt/monitor/client --id my-server --interval 30 http://192.168.1.100:8080/systemmonitor/v1/put http://192.168.1.101:8080/systemmonitor/v1/put > /var/log/monitor/client.log 2>&1 &
```

## 四、常用管理命令

### systemctl 方式

```bash
# 查看服务状态
sudo systemctl status monitor-server
sudo systemctl status monitor-client

# 查看日志
sudo journalctl -u monitor-server -f
sudo journalctl -u monitor-client -f

# 重启服务
sudo systemctl restart monitor-server
sudo systemctl restart monitor-client

# 停止服务
sudo systemctl stop monitor-server
sudo systemctl stop monitor-client

# 取消开机自启
sudo systemctl disable monitor-server
sudo systemctl disable monitor-client
```

### nohup 方式

```bash
# 查看进程
ps aux | grep monitor

# 查看所有 nohup 进程
ps -ef | grep "nohup.*monitor"

# 停止进程
kill <PID>

# 查看日志
tail -f /var/log/monitor/server.log
tail -f /var/log/monitor/client.log
```

## 五、验证部署

### 5.1 检查服务是否运行

```bash
# 检查进程
ps aux | grep monitor

# 检查监听端口
netstat -tlnp | grep 8080
```

### 5.2 访问 Web 界面

打开浏览器访问：

```
http://<服务器IP>:8080/
# 或
http://<服务器IP>:8080/systemmonitor/v1/index.html
```

## 六、注意事项

1. **防火墙**：确保服务器端口（8080）在防火墙中开放
2. **日志轮转**：建议配置 logrotate 防止日志文件过大
3. **权限**：确保程序有权限读取系统信息
4. **网络**：确保客户端能正常访问服务器 URL
5. **nohup vs systemctl**：nohup 适合简单部署，systemctl 适合生产环境（支持日志管理、自动重启等）

## 七、配置日志轮转（可选）

创建 `/etc/logrotate.d/monitor`：

```
/var/log/monitor/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0644 root root
}
```

## 八、完整部署示例

### nohup 方式

```bash
# 1. 创建目录
mkdir -p /opt/monitor /var/log/monitor

# 2. 复制程序
cp server-linux-amd64 /opt/monitor/server
cp client-linux-amd64 /opt/monitor/client
chmod +x /opt/monitor/server /opt/monitor/client

# 3. 启动服务器
nohup /opt/monitor/server --port 8080 --url systemmonitor/v1 > /var/log/monitor/server.log 2>&1 &

# 4. 启动客户端（单个服务器）
nohup /opt/monitor/client --id my-server --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put > /var/log/monitor/client.log 2>&1 &

# 4. 启动客户端（多个服务器）
nohup /opt/monitor/client --id my-server --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put http://192.168.1.101:8080/systemmonitor/v1/put > /var/log/monitor/client.log 2>&1 &

# 5. 检查状态
ps aux | grep monitor
netstat -tlnp | grep 8080
```

### systemctl 方式

```bash
# 1. 创建目录
sudo mkdir -p /opt/monitor /var/log/monitor

# 2. 复制程序
sudo cp server-linux-amd64 /opt/monitor/server
sudo cp client-linux-amd64 /opt/monitor/client
sudo chmod +x /opt/monitor/server /opt/monitor/client

# 3. 创建服务器服务
sudo tee /etc/systemd/system/monitor-server.service > /dev/null <<'EOF'
[Unit]
Description=System Monitor Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/monitor
ExecStart=/opt/monitor/server --port 8080 --url systemmonitor/v1
Restart=always
RestartSec=10
StandardOutput=append:/var/log/monitor/server.log
StandardError=append:/var/log/monitor/server.log

[Install]
WantedBy=multi-user.target
EOF

# 4. 创建客户端服务
sudo tee /etc/systemd/system/monitor-client.service > /dev/null <<'EOF'
[Unit]
Description=System Monitor Client
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/monitor
ExecStart=/opt/monitor/client --id my-server --interval 60 http://192.168.1.100:8080/systemmonitor/v1/put
Restart=always
RestartSec=10
StandardOutput=append:/var/log/monitor/client.log
StandardError=append:/var/log/monitor/client.log

[Install]
WantedBy=multi-user.target
EOF

# 5. 启动服务
sudo systemctl daemon-reload
sudo systemctl enable --now monitor-server monitor-client

# 6. 检查状态
sudo systemctl status monitor-server monitor-client
```
