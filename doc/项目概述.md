# 服务器监控程序项目概述

## 一、项目概述

本项目是一个轻量级的服务器监控系统，支持 Go 语言和 C 语言两种实现。客户端采集本地系统信息并上报，服务器端汇总并通过 Web 页面展示所有客户端状态。

## 二、系统架构

```
┌─────────────┐      JSON POST       ┌─────────────┐
│   Client   │ ──────────────────▶  │   Server    │
│  (采集端)  │    每隔N秒发送一次    │  (展示端)   │
└─────────────┘                      └─────────────┘
       │                                   │
       │                                   ▼
       │                          ┌─────────────┐
       │                          │  Web UI     │
       │                          │  (展示页面) │
       │                          └─────────────┘
       │
       ▼
┌─────────────────────────────────────────────────┐
│  采集内容: hostname, arch, kernel, uptime,      │
│  load, memory, network (name, mac, ipv4, ipv6) │
└─────────────────────────────────────────────────┘
```

## 三、技术方案

### 3.1 Go 版本
- **语言**: Go 1.18+，仅使用内置库
- **通信协议**: HTTP JSON
- **数据存储**: 内存 Map
- **Web 框架**: Go 内置 `net/http`
- **模板渲染**: Go 内置 `html/template`
- **图表**: 使用 Chart.js

### 3.2 C 版本
- **语言**: C (仅支持 Linux)
- **JSON 解析**: parson 库
- **图表**: chart.js
- **多线程**: pthread

## 四、功能特性

### 4.1 客户端功能
- 采集系统信息：hostname, arch, kernel, uptime, load, memory, network
- 网络信息：网卡名称、MAC 地址、IPv4、IPv6
- 支持 Windows (Go) 和 Linux (Go + C)
- 命令行参数指定客户端 ID、发送间隔、服务器 URL
- 支持多个服务器 URL，依次发送

### 4.2 服务器功能
- 接收客户端 JSON 数据
- 记录最后通信时间，超过 2 小时判定离线
- 内存存储数据，包含 128 条 Load 和 Memory 历史记录
- Web 页面展示：左侧客户端列表，右侧详情
- 图表展示：负载历史、内存使用历史
- 自动刷新（可开关）
- 根路径重定向到 Web 页面

## 五、接口设计

### 5.1 客户端上报接口

**端点**: `PUT /{url}/put`  
**方法**: POST  
**Content-Type**: application/json

**请求 JSON 格式**:
```json
{
  "client_id": "abc123",
  "hostname": "server01",
  "arch": "x86_64",
  "kernel": "5.4.0-generic",
  "uptime": 3600,
  "load1": 0.5,
  "load5": 0.3,
  "load15": 0.2,
  "memory_total": 16777216,
  "memory_used": 8388608,
  "timestamp": "2026-03-23T12:00:00+0800",
  "network": [
    {
      "name": "eth0",
      "mac": "00:11:22:33:44:55",
      "ipv4": "192.168.1.100",
      "ipv6": "fe80::211:22ff:fe33:4455"
    }
  ]
}
```

**响应**:
```json
{
  "status": "ok"
}
```

### 5.2 Web 接口

| 端点 | 方法 | 说明 |
|-----|------|-----|
| `/` | GET | 根路径，重定向到 index.html |
| `/{url}/index.html` | GET | Web 页面 |
| `/{url}/data` | GET | JSON 数据 |
| `/{url}/put` | POST | 接收客户端数据 |

## 六、命令行参数

### 6.1 服务器端

| 参数 | 必填 | 默认值 | 说明 |
|-----|------|-------|-----|
| --port, -p | 是 | - | 监听端口号 |
| --url, -u | 是 | - | URL 前缀路径 |
| --quiet | 否 | false | 静默模式，不输出日志 |

**示例**:
```bash
# Go 版本
./server --port 8080 --url systemmonitor/v1

# C 版本
./server --port 8080 --url systemmonitor/v1
```

### 6.2 客户端

| 参数 | 必填 | 默认值 | 说明 |
|-----|------|-------|-----|
| --id | 否 | 随机6位字符串 | 客户端识别码 |
| --interval, -i | 否 | 60 | 发送间隔(秒) |
| URL | 是 | - | 服务器URL(可多个) |

**示例**:
```bash
# Go 版本
./client --id server01 --interval 30 http://192.168.1.1:8080/systemmonitor/v1/put

# C 版本
./client --id server01 http://192.168.1.1:8080/systemmonitor/v1/put
```

## 七、数据结构

### 7.1 客户端信息结构 (Go)

```go
type NetworkInfo struct {
    Name string
    MAC  string
    IPv4 string
    IPv6 string
}

type ClientInfo struct {
    ClientID     string
    Hostname     string
    Arch         string
    Kernel       string
    Uptime       int64
    Load1        float64
    Load5        float64
    Load15       float64
    MemoryTotal  uint64
    MemoryUsed   uint64
    Timestamp    time.Time
    Network      []NetworkInfo
}
```

### 7.2 客户端历史数据 (Go)

```go
type ClientHistory struct {
    LastSeen    time.Time
    IsOnline    bool
    LoadHistory []float64   // 128条
    MemHistory  []uint64    // 128条
    TimeHistory []time.Time // 128条
    LatestInfo  ClientInfo
}
```

## 九、Web 页面功能

### 9.1 页面布局
- 左侧：客户端列表，显示 client_id 和在线状态
- 右侧：选中的客户端详情

### 9.2 详细信息
- 主机名和客户端 ID
- 在线状态
- 最后通信时间（显示"多久之前"）
- IP 地址、架构、内核、运行时间
- 系统负载 (1/5/15 分钟)
- 内存使用情况
- 时间戳
- 网络接口列表

### 9.3 图表
- 负载历史折线图
- 内存使用历史折线图
- 支持鼠标悬停显示详细数值

### 9.4 交互
- 点击左侧列表切换客户端
- 自动刷新（可开关）
- 刷新时保持当前选中客户端

## 十、注意事项

1. 负载和内存历史记录固定为 128 条，FIFO 滚动
2. 客户端识别码支持手动指定或自动生成
3. 服务器不验证客户端身份，信任所有请求
4. 无需持久化，重启后数据清零
5. 时间格式使用 ISO 8601 带时区 (如 `2026-03-23T12:00:00+0800`)
6. C 版本仅支持 Linux
7. 根路径 `/` 自动重定向到 `/{url}/index.html`
