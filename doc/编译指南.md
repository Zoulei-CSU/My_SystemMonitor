# 编译指南

本文档说明如何交叉编译服务器监控程序的客户端和服务器端，支持多种架构和操作系统。

## 一、环境要求

### 1.1 Go 版本
- Go 1.18 或更高版本

### 1.2 C 版本 (Linux only)
- GCC 编译器
- Make 工具
- Linux 系统

## 二、Go 版本编译

### 2.1 环境准备

```bash
go env -w GOPROXY=https://goproxy.cn
# 初始化 Go 模块
go mod init
go mod tidy
```

### 2.2 编译选项说明

| 选项 | 说明 |
|-----|------|
| `-ldflags "-w -s"` | 移除调试信息，显著减小二进制文件大小 |
| `-ldflags "-s -w"` | 同上，顺序不影响 |
| `-trimpath` | 移除路径信息，确保构建可复现 |

### 2.3 编译服务器端

#### Linux

```bash
# AMD64 (推荐，带上-ldflags "-w -s"参数，体积小)
GOOS=linux GOARCH=amd64 go build -ldflags "-w -s" -o server-linux-amd64 ./server

# ARM64 (AArch64)
GOOS=linux GOARCH=arm64 go build -ldflags "-w -s" -o server-linux-arm64 ./server

# ARMv7 (ARM32)
GOOS=linux GOARCH=arm GOARM=7 go build -ldflags "-w -s" -o server-linux-armv7 ./server

# ARMv6 (Raspberry Pi Zero)
GOOS=linux GOARCH=arm GOARM=6 go build -ldflags "-w -s" -o server-linux-armv6 ./server

# 386 (32位)
GOOS=linux GOARCH=386 go build -ldflags "-w -s" -o server-linux-386 ./server
```

#### Windows

```bash
# AMD64
GOOS=windows GOARCH=amd64 go build -ldflags "-w -s" -o server-windows-amd64.exe ./server

# 386
GOOS=windows GOARCH=386 go build -ldflags "-w -s" -o server-windows-386.exe ./server
```

#### macOS

```bash
# AMD64
GOOS=darwin GOARCH=amd64 go build -ldflags "-w -s" -o server-darwin-amd64 ./server

# ARM64 (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build -ldflags "-w -s" -o server-darwin-arm64 ./server
```

### 2.4 编译客户端

#### Linux

```bash
# AMD64
GOOS=linux GOARCH=amd64 go build -ldflags "-w -s" -o client-linux-amd64 ./client

# ARM64
GOOS=linux GOARCH=arm64 go build -ldflags "-w -s" -o client-linux-arm64 ./client

# ARMv7
GOOS=linux GOARCH=arm GOARM=7 go build -ldflags "-w -s" -o client-linux-armv7 ./client
```

#### Windows

```bash
GOOS=windows GOARCH=amd64 go build -ldflags "-w -s" -o client-windows-amd64.exe ./client
GOOS=windows GOARCH=386 go build -ldflags "-w -s" -o client-windows-386.exe ./client
```

#### macOS

```bash
GOOS=darwin GOARCH=amd64 go build -ldflags "-w -s" -o client-darwin-amd64 ./client
GOOS=darwin GOARCH=arm64 go build -ldflags "-w -s" -o client-darwin-arm64 ./client
```

### 2.5 体积优化效果

使用 `-ldflags "-w -s"` 可以显著减小二进制文件大小：

| 平台 | 普通编译 | 优化后 | C版本 |
|-----|---------|--------|-----|
| Linux amd64 | ~10MB | ~7MB | ~200KB |
| Linux aarch64 | ~8MB | ~5MB | ~200KB |
| Windows amd64 | ~9MB | ~6MB | / |

## 三、C 版本编译 (Linux only)

### 3.1 进入 C 目录

```bash
cd C
```

### 3.2 编译

```bash
# 动态链接版本 (依赖系统库)
make

# 静态链接版本 (依赖少，适合嵌入式)
make static
```

### 3.3 Makefile 说明

| 目标 | 说明 |
|-----|------|
| `make` | 动态链接编译 |
| `make static` | 静态链接编译，生成独立二进制 |
| `make clean` | 清理编译产物 |

### 3.4 手动编译

```bash
# 动态链接
gcc -Wall -Wextra -O2 -std=c11 -o server server.c common.c parson.c chart.c -lpthread
gcc -Wall -Wextra -O2 -std=c11 -o client client.c common.c parson.c -lpthread

# 静态链接 (依赖更少)
gcc -Wall -Wextra -O2 -std=c11 -static -o server server.c common.c parson.c chart.c -static-libgcc -Wl,-Bstatic -lpthread
gcc -Wall -Wextra -O2 -std=c11 -static -o client client.c common.c parson.c -static-libgcc -Wl,-Bstatic -lpthread
```

### 3.5 编译参数说明

| 参数 | 说明 |
|-----|------|
| `-Wall -Wextra` | 启用警告 |
| `-O2` | 优化级别 |
| `-std=c11` | C11 标准 |
| `-static` | 静态链接 |
| `-lpthread` | 线程库 |

## 四、一键编译脚本

### 4.1 build-all.sh (Linux/macOS)

```bash
#!/bin/bash

mkdir -p bin

echo "Building Go server..."
GOOS=linux GOARCH=amd64 go build -ldflags "-w -s" -o bin/server-linux-amd64 ./server
GOOS=linux GOARCH=arm64 go build -ldflags "-w -s" -o bin/server-linux-arm64 ./server

echo "Building Go client..."
GOOS=linux GOARCH=amd64 go build -ldflags "-w -s" -o bin/client-linux-amd64 ./client
GOOS=linux GOARCH=arm64 go build -ldflags "-w -s" -o bin/client-linux-arm64 ./client

echo "Build complete!"
ls -la bin/
```

### 4.2 build-all.bat (Windows)

```batch
@echo off
mkdir bin

echo Building Go server...
set GOOS=linux
set GOARCH=amd64
go build -ldflags "-w -s" -o bin\server-linux-amd64 .\server

echo Building Go client...
set GOARCH=amd64
go build -ldflags "-w -s" -o bin\client-linux-amd64 .\client

echo Build complete!
dir bin
```

## 五、常用架构说明

| 架构 | 说明 | 典型设备 |
|-----|------|---------|
| amd64 | 64位 x86 | 大多数 PC 和服务器 |
| arm64 | 64位 ARM | Apple M1/M2, AWS Graviton, 树莓派 4 |
| arm | 32位 ARM | 树莓派 2/3, 旧款 ARM 设备 |
| 386 | 32位 x86 | 旧 PC |

## 六、验证编译结果

```bash
# 查看文件信息

file client-linux-arm64

# 检查是否可执行
ls -la *.exe 2>/dev/null || ls -la server-* client-*
```

## 七、部署

将编译好的二进制文件复制到目标机器：

```bash
# 复制到 Linux 服务器
scp server-linux-amd64 user@192.168.1.100:/opt/monitor/
scp client-linux-amd64 user@192.168.1.101:/opt/monitor/

# 添加执行权限 (Linux)
chmod +x server-linux-amd64
chmod +x client-linux-amd64
```

## 九、版本选择建议

### 9.1 Go 版本 vs C 版本

| 特性 | Go 版本 | C 版本 |
|-----|--------|-------|
| 跨平台 | 支持多平台 | 仅 Linux |
| 依赖 | 无外部依赖 | 需要 pthread |
| 体积 | 较大 (4~8MB) | 较小 (<500KB) |
| 易用性 | 简单 | 需要编译 |
| 静态链接 | 难实现 | 推荐 |

### 9.2 推荐场景

- **桌面/服务器**: 使用 Go 版本，编译简单
- **嵌入式设备**: 使用 C 版本 static，体积小依赖少
- **生产环境**: 两者都可以，根据需求选择
